# Releases এ EXE দেখানো (কীভাবে রিলিজ বানাবেন)

GitHub এর **Releases** পেজে EXE (ZIP) দেখাতে চাইলে একটা **ট্যাগ** পুশ করলেই হবে। ট্যাগ পুশ করলে অটো বিল্ড হবে এবং সেই ভার্সনের রিলিজে EXE জিপ আপলোড হবে।

## ধাপ ১: কোড পুশ করুন

প্রথমে সব কোড main এ পুশ করুন (যেমনটা আগে করছেন):

```bash
cd C:\Users\Adnan\Documents\MAILER
git add -A
git commit -m "Update"
git push origin main
```

## ধাপ ২: ভার্সন ট্যাগ দিন ও পুশ করুন

যে ভার্সন দিয়ে রিলিজ চান (যেমন 1.0.0), সেই অনুযায়ী ট্যাগ বানিয়ে পুশ করুন:

```bash
git tag v1.0.0
git push origin v1.0.0
```

- `v1.0.0` এর জায়গায় যেকোনো ভার্সন দিতে পারেন: `v1.0.1`, `v2.0.0` ইত্যাদি।
- **v** দিয়ে শুরু করতে হবে (যেমন v1.0.0)।

## ধাপ ৩: রিলিজ দেখুন

- ট্যাগ পুশ করলে GitHub Actions অটো বিল্ড চালাবে।
- বিল্ড শেষ হলে **Releases** পেজে নতুন রিলিজ চলে আসবে (ট্যাগের নামে)।
- সেখানে **MailerApp-Desktop-win-x64.zip** ও **MailerApp-Worker-win-x64.zip** ডাউনলোড লিংক থাকবে।

রিপো পেজে গিয়ে **Releases** (ডান পাশে) ক্লিক করলেই সব রিলিজ ও EXE জিপ দেখা যাবে।

## সংক্ষেপে

| করণীয়              | ফলাফল                    |
|----------------------|---------------------------|
| `git tag v1.0.0`     | লোকালে v1.0.0 ট্যাগ তৈরি |
| `git push origin v1.0.0` | ট্যাগ পুশ → Actions বিল্ড → Releases এ EXE জিপ |

পরবর্তী ভার্সনের জন্য নতুন ট্যাগ দিন (যেমন `v1.0.1`) এবং আবার পুশ করুন।
